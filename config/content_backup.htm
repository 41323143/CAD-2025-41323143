w10: Webots 是什麼？有什麼特點？ 如何進行場景模擬？ 如何用 Python 進行控制系統設計？ (Tutorial)

webots是什麼?

webots是一款由 Cyberbotics 開發、跨平台且專業的開源 3D 機器人模擬軟體。
它可用於研究、教育、快速原型製作，以及工業級機器人演算法開發。

你可以在其中建構 3D 世界、加入機器人、加入感測器與致動器，並透過程式語言（Python、C、C++、Java、ROS 等）控制它們的行為與決策。

webots的特點

1. 豐富的機器人與感測器庫

Webots 內建許多常見的機器人模型：

車輪式機器人（如 Pioneer 3-DX, e-puck）

四足/雙足機器人（如 Spot、NAO）

機械手臂（UR5、KUKA…）

飛行器（四旋翼、多旋翼）

也包含各式感測器：

相機（Camera、RGB-D）

雷射（Lidar）

IMU

GPS

距離感測器、觸感感測器等。

2. 完整物理引擎支持

Webots 使用 ODE 物理引擎（並可切換其他引擎），提供：

重力、摩擦、碰撞

關節、平台、驅動馬達

真實物理動力模擬
適用於自走車、機械手臂、類人機器人、飛行器。

3. 可視化3D編輯器

可在 GUI 中輕鬆建立 3D 場景，包括：

地形

建築物

障礙物

道路、標誌

光源、天空環境

拖拉方式建立場景十分直觀。

4. 跨語言控制

你可使用以下語言撰寫控制器：

Python（最常用）

C / C++

Java

MATLAB

ROS (Robot Operating System)

5. 與ROS完整整合

支援 ROS1、ROS2，可搭配：

Nav2

Slam Toolbox

RViz
讓研究者能使用真實機器人軟體套件。

如何進行場景模擬？

Step1 - 建立或開啟一個世界(*.wbt)

打開 Webots → File → New World

或開現有範例，如：

robotics → mobile robots

manipulation → robot arms

Step2 - 在場景中加入物件

按右側 Add → Bot → Obstacle → Environment
可加入：

地面（Floor / Terrain）

桌子、箱子等障礙物

機器人模型

感測器（Lidar、Camera）

光源、天空

Step3 - 加入控制器(controller)

每個機器人都必須綁定一個 controller 程式：

在 Robot 節點中找到 controller 欄位

選擇 new 建立 Python 控制器

Webots 會建立 .py 檔並自動與機器人連結

Step4 - 執行模擬

按 播放鍵 ▶

控制器開始執行，機器人會依你的程式運作

可隨時修改場景、調整物理參數、紀錄影片

如何用 Python 進行控制系統設計？

Webots 提供 Python API，以下示範基本流程：

Step1 - 匯入webots API

from controller import Robot

Step2 - 初始化機器人

robot = Robot()
timestep = int(robot.getBasicTimeStep())

Step3 - 取得馬達與感測器

例如控制輪式機器人:

left_motor = robot.getDevice('left wheel motor')
right_motor = robot.getDevice('right wheel motor')
 
left_motor.setPosition(float('inf'))  # velocity mode
right_motor.setPosition(float('inf'))
 
開啟距離感測器：
ds = robot.getDevice('ds0')
ds.enable(timestep)

Step4 - 主控制迴圈(控制系統核心)

while robot.step(timestep) != -1:
    distance = ds.getValue()
 
    # 簡單避障邏輯
    if distance < 800:
        left_motor.setVelocity(-2.0)
        right_motor.setVelocity(2.0)
    else:
        left_motor.setVelocity(4.0)
        right_motor.setVelocity(4.0)

w14: Webots 的模型有沒有辦法用程式方法建立? 有沒有辦法直接從 CAD 套件中轉出 Webots 的場景模型？

Webots 的模型可以透過程式方式建立，例如使用 PROTO 參數化模型或直接以文字格式產生 world（.wbt）檔案，藉此自動建立或重複使用場景與物件。此外，也可在模擬過程中透過 Supervisor API 動態新增模型。

Webots 亦支援由 CAD 套件匯出模型後匯入使用，常見流程為將 CAD 模型轉成 DAE、STL 或 OBJ 等 Mesh 格式，再於 Webots 中作為場景或機器人外觀模型使用。然而，CAD 匯入僅提供幾何外型，仍需在 Webots 中另外設定碰撞物件、物理參數與關節，才能進行正確模擬。

w17: 如何實作一套 5-bar Linkage Plotter 與 Openduck mini？(Disney Bipedal Robot)
          

本實作先使用 SolveSpace 與 Onshape 進行 5-bar Linkage Plotter 與 Openduck mini（Disney Bipedal Robot）的機構設計與尺寸規劃，完成各連桿與本體之 CAD 建模後，將模型匯出為 STL 或 DAE 等格式，再匯入 Webots 作為機器人與場景的外觀模型。

在 Webots 中，針對匯入的 CAD 模型重新建立 Solid 與 HingeJoint 結構，設定各關節的馬達控制、碰撞物件（boundingObject）與物理參數（Physics）。5-bar Linkage Plotter 透過運動學計算將繪圖軌跡轉換為兩主動關節角度，以控制末端點完成繪圖；Openduck mini 則透過雙足關節協調與步態規劃，實現基本行走與平衡動作。

w18: 課程總結

這門課從零組件設計一路做到模擬與控制，讓我第一次實際接觸到機構設計與機器人模擬整合的流程。一開始在4連桿的部分，使用 SolveSpace 與 Onshape 畫連桿與零件，過程中比較能理解平面機構的運動關係；而 Openduck mini 是成員畫的，主要用 Onshape 建模，結構較複雜，在將模型匯入 Webots 後，才發現和單純畫圖差很多。模型匯進來之後如果沒有設定關節、碰撞與物理參數，機構是無法正常運動的，透過實際操作 Webots，逐步加入馬達與關節控制，才慢慢理解模擬環境中動態行為是怎麼產生的，後續使用 Python 進行控制時，讓我把前面學到的機構概念實際用在控制上，這門課讓我不是只停留在畫圖或寫程式，而是能把設計、模擬與控制串在一起，對之後做專題或相關課程都有幫助。
